# AGENTS.md

## Назначение репозитория

Репозиторий предназначен для подготовки к экзамену по компьютерному зрению. Основная цель - поддерживать два набора материалов: конспекты презентаций курса и развёрнутые ответы на экзаменационные вопросы.

Все материалы пишутся на русском языке. Исходные учебные файлы считаются источниками и не должны изменяться без явной просьбы пользователя.

## Полная карта репозитория

Этот раздел перечисляет текущие папки и файлы репозитория с кратким назначением. Если структура меняется, обновляйте эту карту вместе с индексами.

## Рабочие команды

Запускать из корня репозитория.

```powershell
uv run --with pypdf python .\tools\generate_presentation_notes.py
```

```powershell
uv run python .\tools\validate_notes.py
```

`generate_presentation_notes.py` требует `pypdf`, поэтому команда запускает его через `uv run --with pypdf`. Если в репозиторий позже добавят `pyproject.toml` с зависимостями, `--with pypdf` можно заменить на обычный `uv run python ...`.

Успешная финальная проверка должна выглядеть так:

```text
PDFs: 24
Presentation notes: 24
Questions: 31
Question notes: 31
Validation errors: 0
```

## Правила для ИИ-агентов

- Не изменять исходные `PDF`, `.ipynb` и `Вопросы.md`, если пользователь явно не попросил.
- Все новые и изменённые Markdown-файлы сохранять в UTF-8.
- Для чтения кириллических файлов в PowerShell использовать `-Encoding UTF8`.
- Не удалять `Конспекты_презентаций/_assets/`: изображения нужны для визуальных презентаций.
- При обновлении ответов сохранять структуру разделов и экзаменационную глубину.
- При добавлении нового PDF обновить карту в `tools/generate_presentation_notes.py`, затем перегенерировать конспекты и запустить `validate_notes.py`.
- При добавлении или изменении вопроса обновить соответствующий `NN_*.md`, затем запустить `validate_notes.py`.
- Если меняется только содержание одного конспекта, не нужно перегенерировать весь корпус без необходимости: это может перезаписать ручные улучшения.
- Внешние источники использовать для проверки и заполнения пробелов, но курс остаётся основным источником.

## Качество содержания

Для конспектов презентаций важно не просто пересказывать извлечённый текст, а восстанавливать смысл слайдов: определения, архитектуры, формулы, ограничения методов, связи с вопросами и практикой.

Для ответов на вопросы важно, чтобы файл был самостоятельным: студент должен открыть один `NN_*.md` и получить полный ответ для устного экзамена без необходимости читать все презентации.

## Текущие ограничения

- `pypdf` иногда печатает предупреждения вида `Ignoring wrong pointing object ...`; сами конспекты при этом могут генерироваться корректно.
- Полноценного OCR/рендера PDF может не быть в окружении. Поэтому визуальные PDF обработаны через извлечение встроенных изображений и экспертное описание.
