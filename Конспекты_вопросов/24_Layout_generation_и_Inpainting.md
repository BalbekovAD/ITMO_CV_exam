# 24. Генерация layout расположения объектов и вписывание части изображения

## Краткий ответ

Layout generation решает задачу построения структурного плана сцены: какие объекты присутствуют, где они расположены, какого они размера и иногда как связаны между собой. Такой layout может задаваться bounding boxes, масками, сценным графом или последовательностью токенов. Вписывание части изображения обычно относится к inpainting/outpainting: модель заполняет маскированную область так, чтобы она была семантически правдоподобна и согласована с окружением по цвету, освещению, перспективе и текстуре.

## Основные понятия

- Layout: абстрактное описание композиции сцены.
- Bounding box layout: набор прямоугольников с классами объектов.
- Scene graph: граф объектов и отношений, например "человек держит чашку".
- Semantic layout: карта классов по пикселям.
- Inpainting: заполнение пропущенной области внутри изображения.
- Outpainting: расширение изображения за исходные границы.
- Compositing: вставка объекта или фрагмента в существующее изображение.
- Context consistency: согласованность с окружающими пикселями.
- Object relation constraints: ограничения на взаимное положение объектов.

## Алгоритмы, архитектуры и формулы

Классическая постановка layout generation:

$$
p(L \mid c), \quad L = \{(\operatorname{class}_i, \operatorname{box}_i)\}_{i=1..N}
$$

Условие `c` может быть текстом, набором объектов, сценным графом или категорией сцены. Затем изображение генерируется как:

$$
p(x \mid L, c)
$$

Подходы к layout generation:

- Авторегрессионные модели: layout представляется последовательностью токенов `class, x, y, w, h`; модель генерирует токены по очереди.
- Transformer-подходы: учитывают отношения между объектами через self-attention.
- Диффузионные модели layout: добавляют шум к координатам/маскам и учатся восстанавливать структуру.
- Graph-based подходы: используют сценный граф и преобразуют отношения в spatial layout.
- Layout-to-image: сначала генерируется layout, потом изображение с помощью GAN или diffusion.

Для inpainting в диффузии часто используется маска `m`:

$$
\begin{aligned}
x_{\text{known}} &= m x_{\text{original}}, \\
x_{\text{unknown}} &= (1 - m)x_{\text{generated}}, \\
x_{\text{result}} &= x_{\text{known}} + x_{\text{unknown}}.
\end{aligned}
$$

На каждом шаге сэмплирования известная часть может принудительно возвращаться к исходному изображению с соответствующим уровнем шума, а неизвестная часть генерируется моделью. Это позволяет сохранить неизменяемые области.

Архитектуры inpainting:

- CNN encoder-decoder с partial/gated convolutions;
- GAN-based inpainting с adversarial loss;
- diffusion inpainting с маской и текстовым условием;
- image-conditioned diffusion для редактирования по тексту.

Функции потерь для классического inpainting могут включать:

- reconstruction loss в маске и вне маски;
- perceptual loss по признакам VGG;
- style loss через матрицы Грама;
- adversarial loss;
- total variation loss для гладкости.

## Сравнения, плюсы и минусы

Layout как bounding boxes:

- Плюсы: компактно, удобно для детекции и композиции.
- Минусы: плохо описывает форму и точные границы объектов.

Semantic mask:

- Плюсы: точнее задает форму и классы областей.
- Минусы: дороже размечать, сложнее генерировать.

Scene graph:

- Плюсы: хорошо описывает отношения.
- Минусы: требуется преобразование отношений в геометрию.

Diffusion inpainting:

- Плюсы: высокое качество, хорошо использует текстовое условие, может дорисовывать сложные детали.
- Минусы: может менять смысл объекта, нарушать идентичность, давать нестабильные результаты при плохой маске.

## Типичные ошибки

- Считать layout готовым изображением. Layout - это структурное описание, а не визуальный результат.
- Ожидать от bounding boxes точной формы объекта.
- Делать слишком маленькую маску для inpainting: модели нужен контекст и пространство для согласованного перехода.
- Игнорировать перспективу и освещение при вставке объекта.
- Путать inpainting с super-resolution: первая задача заполняет неизвестную область, вторая повышает детализацию известного изображения.
