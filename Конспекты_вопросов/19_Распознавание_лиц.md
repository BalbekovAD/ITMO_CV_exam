# 19. Распознавание лиц

## Краткий ответ

Распознавание лиц - задача определения личности по изображению лица или сравнения двух лиц. Типовой pipeline включает детекцию лица, выравнивание по ключевым точкам, извлечение embedding нейросетью и сравнение embeddings по cosine similarity или евклидову расстоянию. Современные системы обучаются так, чтобы изображения одного человека имели близкие embeddings, а разных людей - далекие.

## Основные понятия

- **Face detection**: найти bounding box лица.
- **Face alignment**: привести лицо к стандартной ориентации по глазам, носу, рту.
- **Face embedding**: компактный вектор признаков личности.
- **Verification**: ответить, один и тот же человек на двух изображениях или нет.
- **Identification**: выбрать личность из базы.
- **Open-set recognition**: человек может отсутствовать в базе.
- **Metric learning**: обучение пространства признаков с расстояниями.
- **Liveness / anti-spoofing**: проверка, что перед камерой живой человек, а не фото/маска/экран.

## Алгоритмы и архитектуры

Классический pipeline:

```text
image -> face detector -> landmarks -> alignment -> CNN -> embedding -> comparison
```

Ранние классические методы:

- Eigenfaces: PCA на изображениях лиц.
- Fisherfaces: LDA для разделения классов.
- LBP/HOG-признаки: локальные текстурные описания.

Современные методы используют CNN/Transformer backbone: ResNet, Inception-ResNet, MobileFaceNet, ViT-like модели. Главная часть - loss для discriminative embeddings.

**Siamese network + contrastive loss**:

$$
L = y d^2 + (1-y)\max(0, \operatorname{margin} - d)^2
$$

Где $y=1$ для одной личности, $d$ - расстояние между embeddings.

**Triplet loss**:

$$
L = \max(0, d(\operatorname{anchor}, \operatorname{positive}) - d(\operatorname{anchor}, \operatorname{negative}) + \operatorname{margin})
$$

Он сближает anchor-positive и отдаляет anchor-negative.

**ArcFace** - одна из канонических современных функций потерь. Она добавляет angular margin в softmax-классификацию:

$$
L = -\log
\frac{\exp(s\cos(\theta_y + m))}
{\exp(s\cos(\theta_y + m)) + \sum_{j \ne y}\exp(s\cos(\theta_j))}
$$

Идея: не просто правильно классифицировать, а разделить классы в нормированном угловом пространстве.

## Метрики

- **Accuracy на verification pairs**: доля правильных решений.
- **ROC/AUC**: trade-off между TPR и FPR.
- **FAR/FRR**: false accept rate и false reject rate.
- **EER**: точка, где $FAR = FRR$.
- **CMC/top-k accuracy**: для identification.

В практической системе threshold выбирают по допустимому FAR: например, для банковской биометрии ложный допуск критичнее, чем ложный отказ.

## Сравнения, плюсы и минусы

Verification проще, чем identification: нужно сравнить пару, а не искать по большой базе. Closed-set identification проще open-set, потому что предполагает, что человек точно есть среди классов.

Face recognition удобен, потому что лицо легко снять камерой, но уязвим к изменению освещения, позы, возраста, окклюзий, качества камеры, domain shift и spoofing-атакам. Также есть важные этические и правовые ограничения: приватность, согласие, bias по демографическим группам.

## Типичные ошибки

- Путать детекцию лица и распознавание личности.
- Сравнивать embeddings без нормализации, если модель обучалась под cosine/angular distance.
- Использовать classification head по train identities как готовую систему для новых людей без embedding-подхода.
- Не учитывать threshold и FAR/FRR в реальном применении.
- Игнорировать alignment: даже хорошая embedding-модель страдает от неверно вырезанных лиц.
- Забывать про anti-spoofing и юридические ограничения.
