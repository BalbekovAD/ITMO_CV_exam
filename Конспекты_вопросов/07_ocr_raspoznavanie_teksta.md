# 7. Распознавание текста (OCR)

## Краткий ответ

OCR - задача извлечения текста из изображения документа, сцены или видеокадра. Полный pipeline обычно включает предобработку изображения, обнаружение текстовых областей, распознавание символов/строк и постобработку языковой моделью или словарями. Современные OCR-системы часто используют CNN/Transformer encoder для визуальных признаков и CTC или seq2seq decoder для получения последовательности символов.

## Основные понятия

- Text detection: найти области текста.
- Text recognition: прочитать текст внутри найденной области.
- Layout analysis: понять структуру документа: колонки, таблицы, заголовки.
- CTC: Connectionist Temporal Classification, loss для последовательностей без посимвольной разметки выравнивания.
- CER/WER: character error rate и word error rate.
- Scene text: текст в естественных сценах, обычно сложнее сканов.

## Алгоритмы, архитектуры, формулы

Классический OCR pipeline:

1. grayscale, binarization, denoising;
2. deskew - исправление наклона;
3. segmentation: строки, слова, символы;
4. распознавание символов;
5. spell-check и языковая постобработка.

Классические методы:

- thresholding, Otsu/adaptive threshold;
- connected components;
- projection profiles для строк;
- HOG + SVM для символов;
- шаблонное сопоставление.

Современный text detection:

- EAST: dense prediction геометрии текстовых областей;
- CTPN: proposals для текстовых строк;
- DBNet: differentiable binarization;
- detector на базе Faster R-CNN/YOLO/segmentation.

Современный text recognition:

- CNN извлекает признаки;
- RNN/LSTM или Transformer моделирует последовательность;
- CTC декодирует строку без точного alignment;
- attention/seq2seq декодер генерирует символы авторегрессионно.

CTC суммирует вероятности всех допустимых выравниваний `pi`, которые сворачиваются в целевую строку `y`:

$$
\begin{aligned}
P(y \mid x) &= \sum_{\pi \in B^{-1}(y)} P(\pi \mid x), \\
L_{CTC} &= -\log P(y \mid x).
\end{aligned}
$$

## Сравнения, плюсы и минусы

Классический OCR:

- плюс: хорошо работает на чистых сканах с простыми шрифтами;
- плюс: интерпретируемый pipeline;
- минус: плохо переносится на сцены, перспективу, нестандартные шрифты.

Нейросетевой OCR:

- плюс: устойчивее к вариативности шрифтов, фона и перспективы;
- плюс: можно распознавать строку целиком без сегментации на символы;
- минус: нужны размеченные данные;
- минус: ошибки могут быть систематическими для редких символов и языков.

CTC:

- плюс: не нужна разметка границ символов;
- минус: хуже работает при сильной языковой неоднозначности без декодера.

Seq2seq attention:

- плюс: гибкое моделирование контекста;
- минус: может галлюцинировать текст.

## Связь с практикой и материалами курса

Тема соответствует презентации `Методы распознования текста.pdf`. OCR связан с детекцией, сегментацией и sequence modeling. В реальных задачах важно разделять OCR документов и OCR естественных сцен: для документов важны layout/table extraction, для сцен - устойчивость к перспективе, бликам, малому размеру текста.

## Типичные ошибки

- Считать OCR только классификацией отдельных символов.
- Игнорировать этап text detection.
- Не исправлять поворот, перспективу и контраст документа.
- Оценивать качество только по accuracy символов, забывая CER/WER.
- Использовать словарь там, где возможны номера, коды и имена собственные.
