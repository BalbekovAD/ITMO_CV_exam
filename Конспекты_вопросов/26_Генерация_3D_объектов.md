# 26. Генерация 3D объектов

## Краткий ответ

Генерация 3D объектов создает форму, геометрию и иногда текстуру объекта в трехмерном пространстве. Главное отличие от 2D-генерации - необходимость согласованности вида с разных камер. Представления 3D включают voxel grids, point clouds, meshes, implicit fields, NeRF и 3D Gaussian Splatting. Современные методы часто используют 2D diffusion models как сильный prior: текст задает объект, 2D-модель оценивает вид, а 3D-представление оптимизируется так, чтобы его рендеры соответствовали тексту.

## Основные понятия

- Mesh: вершины, ребра и полигоны, обычно треугольники.
- Point cloud: набор 3D-точек, иногда с цветом/нормалями.
- Voxel grid: трехмерная сетка занятости или плотности.
- Implicit representation: функция, например occupancy или signed distance field.
- NeRF: нейронное поле радиации, задающее плотность и цвет в точках пространства.
- 3D Gaussian Splatting: представление сцены набором гауссиан с цветом, прозрачностью и ковариацией.
- Differentiable rendering: рендеринг, через который можно распространять градиент.
- SDS: score distillation sampling, оптимизация 3D через 2D diffusion prior.
- Multi-view consistency: согласованность объекта при разных ракурсах.

## Алгоритмы, архитектуры и формулы

Основные 3D-представления:

1. Voxels:
   - Просты, похожи на 3D-изображение.
   - Память растет как $O(N^3)$, поэтому высокое разрешение дорого.

2. Point clouds:
   - Компактны для поверхности.
   - Не задают явную топологию и поверхность.

3. Meshes:
   - Удобны для графики, игр, CAD и рендеринга.
   - Сложно генерировать корректную топологию.

4. Implicit fields:
   - Функция $f_\theta(x, y, z)$ возвращает occupancy, SDF или плотность.
   - Хорошо описывает гладкие поверхности.

5. NeRF:
   - Модель $F_\theta(\operatorname{position}, \operatorname{direction}) \to (\sigma, \operatorname{color})$.
   - Изображение получается объемным рендерингом вдоль лучей.

Упрощенная формула объемного рендеринга:

$$
\begin{aligned}
C(r) &= \sum_i T_i\alpha_i c_i, \\
T_i &= \prod_{j<i}(1 - \alpha_j).
\end{aligned}
$$

где $\alpha_i$ зависит от плотности в точке на луче, $c_i$ - цвет.

Text-to-3D через SDS:

1. Инициализируется 3D-представление, например NeRF или mesh.
2. Случайно выбирается камера.
3. Рендерится изображение объекта.
4. 2D diffusion model оценивает, насколько рендер соответствует тексту.
5. Градиент обновляет 3D-представление.

Схема:

```text
text prompt -> diffusion prior
3D representation -> differentiable render -> image
diffusion score -> gradient -> update 3D
```

SDS не обучает новую diffusion-модель; он использует уже обученный 2D score model как источник градиента для параметров 3D-представления. Практически это может давать Janus-проблему: объект выглядит правдоподобно с отдельных ракурсов, но разные стороны оказываются семантически несогласованными.

Другой класс методов - feed-forward 3D generation: модель сразу предсказывает 3D по одному изображению или тексту, часто с использованием трансформеров и multi-view диффузии.

## Сравнения, плюсы и минусы

Voxels:

- Плюсы: простая структура данных.
- Минусы: плохая масштабируемость по разрешению.

Point clouds:

- Плюсы: экономичны, легко получать со сканеров.
- Минусы: нет поверхности, сложно текстурировать.

Meshes:

- Плюсы: промышленный стандарт для игр и 3D-редакторов.
- Минусы: топологические ошибки, дырки, самопересечения.

NeRF:

- Плюсы: фотореалистичные виды, непрерывное представление.
- Минусы: труднее извлекать редактируемую mesh-геометрию, медленный рендеринг в базовом виде.

3D Gaussian Splatting:

- Плюсы: быстрый рендеринг, хорошее качество реконструкции сцен.
- Минусы: менее естественное CAD-представление, редактирование формы не всегда удобно.

## Типичные ошибки

- Считать 3D-генерацию просто набором независимых 2D-видов.
- Не различать реконструкцию 3D по изображениям и генерацию нового объекта.
- Игнорировать multi-view consistency.
- Не замечать Janus/multi-face артефакты в text-to-3D, когда 2D prior хорошо оценивает отдельные рендеры, но не гарантирует единую 3D-форму.
- Путать mesh, voxel, point cloud и NeRF как взаимозаменяемые форматы.
- Оценивать только красивый рендер, не проверяя пригодность модели для редактирования или экспорта.
